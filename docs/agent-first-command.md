# الأمر الأول للوكيل

انسخ النص التالي للوكيل داخل VS Code أو Codex أو Claude Code:

```text
أنت تعمل على مشروع تطبيق ويب ثابت اسمه "بطاقة عيدك".

الهدف: بناء نسخة أولى MVP لتطبيق تهنئة عيد يعمل على GitHub Pages، يسمح للمستخدم باختيار قالب بطاقة، تعديل اسم المهدى إليه وعبارة التهنئة واسم المرسل، معاينة البطاقة مباشرة، ثم تحميلها كصورة PNG. لا تبنِ Backend ولا قاعدة بيانات ولا تسجيل دخول.

اقرأ الملفات التالية أولًا ولا تعدّل أي شيء قبل فهمها:
- CLAUDE.md
- README.md
- docs/project-brief.md
- docs/product-requirements.md
- docs/ui-rules.md
- docs/data-contracts.md
- docs/architecture.md
- docs/tasks.md
- docs/status.md
- docs/known-issues.md
- docs/design-workflow.md

المطلوب الآن:
1. افحص بنية المشروع الحالية.
2. إن لم تكن ملفات التطبيق موجودة، أنشئ نسخة أولى بسيطة في ملف index.html فقط.
3. ابنِ واجهة عربية RTL مناسبة للجوال.
4. أضف 3 قوالب مبدئية للبطاقة داخل الكود باستخدام HTML/CSS/SVG قدر الإمكان:
   - فاخر ذهبي
   - ناعم فاتح
   - شبابي مرح
5. أضف حقول تعديل:
   - اسم المهدى إليه
   - عبارة التهنئة
   - اسم المرسل
6. اجعل المعاينة تتحدث مباشرة عند الكتابة.
7. أضف زر تحميل PNG للبطاقة.
8. أضف زر مشاركة يحاول مشاركة الصورة مع نص يحتوي اسم المهدى إليه، مع fallback للتحميل ونسخ النص.
9. لا تضف ميزات خارج النطاق مثل GIF أو MP4 أو Backend.
10. بعد التنفيذ، حدّث docs/status.md وعلّم المهام المنجزة في docs/tasks.md.

ابدأ بتقديم خطة تنفيذ قصيرة جدًا، ثم نفّذ التعديل على الملفات. لا تكتب الكود كاملًا في الرد إلا إذا احتجت توضيحًا صغيرًا.
```
