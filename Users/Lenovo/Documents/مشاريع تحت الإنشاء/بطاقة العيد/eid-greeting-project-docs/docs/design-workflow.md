# طريقة التعامل مع التصاميم

## القرار المقترح

في النسخة الأولى، لا تجعل التصاميم صورًا جاهزة بالكامل من وكيل آخر. الأفضل أن يصمم الوكيل القوالب كـ HTML/CSS/SVG داخل المشروع.

## لماذا؟

- أسهل في تعديل النصوص.
- أسهل في التصدير إلى PNG.
- أخف في التحميل.
- لا توجد مشاكل حقوق صور.
- يمكن تغيير الألوان والخطوط برمجيًا.
- مناسب أكثر لـ GitHub Pages.

## متى أستخدم وكيل تصميم آخر؟

استخدم وكيل تصميم أو توليد صور في حالتين فقط:

1. لإنتاج خلفيات زخرفية بسيطة ومحلية داخل assets.
2. لاستلهام الهوية البصرية والألوان والزخارف.

## الصيغة المفضلة لأصول التصميم

الأفضل بالترتيب:

1. CSS gradients + SVG ornaments.
2. SVG assets محلية.
3. PNG backgrounds محلية بدقة عالية.
4. صور خارجية من الإنترنت: غير مفضلة.

## مواصفات أي أصل تصميم خارجي

إذا تم استخدام خلفية أو زخرفة من وكيل آخر، يجب أن تكون:

- بدون نصوص مدمجة.
- مناسبة لمقاس 1920×1080.
- خفيفة الحجم قدر الإمكان.
- محفوظة داخل assets/templates.
- قابلة للاستخدام تجاريًا أو مولدة خصيصًا للمشروع.

## توصية عملية

ابدأ بثلاثة قوالب برمجية:

- فاخر ذهبي.
- ناعم فاتح.
- شبابي مرح.

بعد نجاح التصدير والمشاركة، حسّن جمال التصاميم تدريجيًا.
