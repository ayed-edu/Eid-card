# حالة المشروع

## المرحلة الحالية

تم بناء نسخة MVP أولى في ملف `index.html` واحد.

## القرارات المعتمدة

- التطبيق هدية للمتابعين في العيد.
- التطبيق يعمل من GitHub Pages.
- لا يوجد Backend.
- المخرج الأساسي PNG.
- JPG خيار ثانوي لاحق.
- لا يتم تنفيذ GIF أو فيديو في النسخة الأولى.
- النص المشارك يحتوي اسم المهدى إليه.
- التصاميم المبدئية يمكن أن تكون HTML/CSS/SVG داخل المشروع.
- التصدير الحالي يستخدم Canvas API مباشر بدون مكتبات خارجية.

## آخر إنجاز

تم تنفيذ فحص ما قبل النشر على GitHub Pages بدون إضافة مميزات جديدة.

تم التحقق من:

- عمل التطبيق من `index.html` مباشرة.
- وجود الخلفيات المحلية المطلوبة داخل `assets/templates`:
  - `eid-luxury-dark.png`
  - `eid-cream-gold.png`
  - `eid-soft-green.png`
- صحة المسارات النسبية للصور وملف بيانات التصدير بحيث تناسب GitHub Pages.
- عدم وجود مسارات محلية مثل `C:\` أو `localhost` داخل ملفات المشروع المطلوبة.
- عدم وجود شعار أو حساب أو توقيع أو عبارة حقوق داخل ملف التطبيق أو مسار التصدير.
- تحميل الخطوط قبل تصدير PNG عبر `document.fonts.ready` بدون كسر عملية التصدير.
- عمل تصدير PNG عند فتح الصفحة عبر خادم HTTP محلي يحاكي GitHub Pages.
- عمل تصدير PNG عند فتح `index.html` مباشرة عبر `file://` بعد إضافة `assets/templates/template-image-data.js` كنسخة بيانات محلية للصور وقت التصدير.
- عمل الأزرار الأساسية: اختيار القالب، تعديل الاسم، تعديل العبارة، تعديل اسم المرسل، المعاينة، تحميل PNG، والمشاركة/fallback.
- عدم وجود تجاوز أفقي في عرض جوال 390px، وظهور زر التحميل بحجم واضح.

ملاحظة: ملف `assets/templates/template-image-data.js` يولّد بيانات صور محلية للتصدير حتى لا يتأثر Canvas بقيود قراءة الصور عند فتح الملف مباشرة، ويجب تحديثه إذا تغيّرت ملفات الخلفيات.

## إنجاز سابق: دمج الخلفيات

تم دمج الخلفيات الجاهزة الموجودة داخل `assets/templates` كقوالب التطبيق الأساسية بدل القوالب المرسومة بالكود.

تم تنفيذ:

- استخدام `eid-luxury-dark.png` و`eid-cream-gold.png` و`eid-soft-green.png` كخلفيات مباشرة للمعاينة.
- ضبط نسبة أبعاد كل بطاقة حسب أبعاد صورتها الأصلية بدون تمديد أو تشويه.
- إزالة عنوان البطاقة المرسوم بالكود لأن عبارة العيد موجودة ثابتة داخل الخلفيات.
- الإبقاء فقط على النصوص القابلة للتعديل فوق المساحات الفارغة: اسم المهدى إليه، عبارة التهنئة، واسم المرسل.
- ضبط مواضع النصوص لكل قالب بإحداثيات نسبية في CSS وCanvas.
- تصدير PNG بأبعاد الخلفية الأصلية لكل قالب:
  - `eid-luxury-dark`: 1060×1484
  - `eid-cream-gold`: 1054×1492
  - `eid-soft-green`: 1075×1463
- الحفاظ على البطاقة والصورة المصدّرة بدون شعار أو حساب أو توقيع أو علامة مائية.
- إضافة قيم افتراضية مختلفة لكل قالب: قالب باسم رجل وقالبان بأسماء نساء.
- تحديث حقول النص تلقائيًا عند اختيار القالب طالما لم يكتب المستخدم تعديلًا يدويًا.
- الحفاظ على تعديلات المستخدم اليدوية عند التنقل بين القوالب بعد أول كتابة.
- ضبط مواضع النصوص النهائية فوق الخلفيات الثلاث بحيث يبقى الاسم في المساحة الأولى، والعبارة في المساحة الوسطى، واسم المرسل في المساحة السفلية بعد كلمة "من".
- إضافة هوامش داخلية وحساب تصغير خط عند التصدير حتى لا تلامس النصوص حدود الإطارات.
- ضبط عبارة التهنئة لتظهر بحد أقصى 3 أسطر في المعاينة والتصدير.

تم التحقق محليًا عبر Chrome headless من اختيار القوالب، تحديث المعاينة، رسم النص العربي داخل PNG، تصدير القوالب الثلاثة، وعدم وجود تجاوز أفقي على عرض جوال 390px مع ظهور زر التحميل بوضوح.

## إنجاز سابق: بناء القوالب

تمت إعادة بناء القوالب بصريًا لتقترب من المرجع المرفق بدون نسخ حرفي وبدون استخدام صور خارجية.

تم تنفيذ:

- قالب فاخر داكن بخلفية كحلية/خضراء داكنة، قوس إسلامي كبير، هلال ونجوم، فوانيس، زخارف ذهبية، ولوحة اسم ذهبية.
- قالب أبيض/عاجي ذهبي بإطار رقيق وزخارف زوايا ومساحة مركزية مرتبة.
- قالب حديث ناعم بألوان بيج/أخضر/ذهبي، زخارف خفيفة، فوانيس بسيطة، وخروف لطيف غير مسيطر.
- إزالة أي توقيع أو اسم حساب أو علامة مائية من المعاينة ومن PNG المصدّر.
- توحيد خطوط البطاقة على IBM Plex Sans Arabic، مع جعل اسم المهدى إليه عريضًا والعبارة واسم المرسل بوزن عادي.
- انتظار تحميل الخطوط عبر `document.fonts.ready` قبل تصدير PNG.

تم تحويل اتجاه القوالب والنصوص الافتراضية إلى عيد الأضحى المبارك تحديدًا.

تم تعديل:

- القالب الأول إلى تصميم أضحى فاخر وروحاني بخلفية كحلية/خضراء وزخارف ذهبية وشكل تجريدي للكعبة وقوس إسلامي.
- القالب الثاني إلى تصميم عائلي ناعم بألوان بيج وأخضر هادئ وزخارف بسيطة.
- القالب الثالث إلى تصميم حديث وخفيف بتدرجات مريحة وزخارف هندسية ورمز خروف صغير غير مسيطر.
- النصوص الافتراضية إلى عبارات مناسبة للأضحى مثل: عيد أضحى مبارك، وكل عام وأنتم بخير.
- نص المشاركة ليبدأ بـ: أضحى مبارك يا {اسم المهدى إليه}.

تمت مراجعة نسخة MVP الأولى وتحسين استقرارها بدون إضافة مميزات جديدة.

تم إصلاح:

- التفاف الكلمات الطويلة جدًا داخل PNG عند عدم وجود مسافات.
- نسخ نص المشاركة عند فشل Clipboard API عبر fallback داخلي.
- إخفاء سطر المرسل في المعاينة إذا تُرك فارغًا.
- تأخير تحرير رابط ملف PNG بعد بدء التحميل لتحسين توافق المتصفحات.

تم التحقق محليًا عبر Chrome headless من:

- تحديث المعاينة عند إدخال اسم عربي.
- إنتاج PNG واضح من القالب المختار.
- ظهور النص العربي داخل الصورة المصدرة.
- اختيار القوالب الثلاثة وربط كل قالب بالمعاينة والتصدير.
- بديل المشاركة: تحميل الصورة ونسخ نص التهنئة.
- عدم وجود تجاوز أفقي في واجهة الجوال عند عرض 390px.
- استمرار تصدير القوالب الثلاثة كصور PNG بعد تعديل اتجاه عيد الأضحى.
- ظهور الخطوط داخل المعاينة وداخل PNG على Chrome Desktop headless.
- عدم وجود توقيع أو حساب داخل البطاقة المصدرة.
- عدم وجود نص توقيع أو حساب ثابت في واجهة البطاقة أو مسار رسم Canvas.

## الخطوة التالية

رفع المشروع إلى GitHub وتفعيل GitHub Pages، ثم اختبار الرابط المنشور على Chrome Android وSafari iPhone إن أمكن.
